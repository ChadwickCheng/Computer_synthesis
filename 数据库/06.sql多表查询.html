🔗 SQL多表查询详解与示例

🔗 连接查询（JOIN）

1️⃣ 等值连接（两表）

-- Student 与 SC 按 Sno 等值连接（WHERE写法） SELECT Student.Sno, Sname, Cno, Grade FROM Student, SC WHERE Student.Sno = SC.Sno; -- JOIN写法（等价） SELECT Student.Sno, Sname, Cno, Grade FROM Student JOIN SC ON Student.Sno = SC.Sno;
💡 等值连接：连接条件用 =，结果中两表的 Sno 列都会保留（不去重复列）
💡 同名列需加表名前缀区分，如 Student.Sno
⚠️ 等值连接 ≠ 自然连接：自然连接自动合并同名列，等值连接保留两个同名列

2️⃣ 三表连接

-- Student、SC、Course 三表连接 SELECT Student.Sno, Sname, Course.Cno, Cname, Grade FROM Student, SC, Course WHERE Student.Sno = SC.Sno AND SC.Cno = Course.Cno;
💡 n 张表连接需要 n-1 个连接条件
💡 三表连接时，两个条件都不能少，否则产生笛卡尔积

3️⃣ 自身连接（Self Join）

-- 查询比"张三"年龄大的所有学生 SELECT B.Sname, B.Age FROM Student AS A, Student AS B WHERE A.Sname = '张三' AND B.Age > A.Age; -- 查询同龄的学生对（A.Sno < B.Sno 避免重复） SELECT A.Sname, B.Sname, A.Age FROM Student AS A, Student AS B WHERE A.Age = B.Age AND A.Sno < B.Sno;
💡 自身连接：同一张表起两个不同别名，把它当成两张独立的表
💡 必须用别名（A、B）区分，否则列名引用会报错

4️⃣ 外连接（LEFT JOIN）

-- 查询所有学生及其选课（没选课的学生也显示） SELECT Student.Sno, Sname, Cno, Grade FROM Student LEFT JOIN SC ON Student.Sno = SC.Sno;
💡 LEFT JOIN：左表全部保留，右表无匹配则填 NULL
💡 内连接（JOIN）只保留两表都有的行；左外连接保留左表所有行
🔍 子查询（SubQuery）

5️⃣ 不相关子查询（由内向外执行）

-- 查询选了"数据库"课程的学生姓名 -- 第①步（内层）：找到"数据库"的课程号 -- 第②步（外层）：在SC中找选了该课的Sno，再查Student SELECT Sname FROM Student WHERE Sno IN ( SELECT Sno FROM SC WHERE Cno IN ( SELECT Cno FROM Course WHERE Cname = '数据库' ) );
💡 不相关子查询：子查询独立执行，不引用外层表的列，由内向外逐层执行
⚠️ 子查询中不能使用 ORDER BY（ORDER BY 只能用于最终结果）

② 中层：在SC中找对应Sno

③ 外层：查Student姓名

6️⃣ ANY 与 ALL

-- ANY：C01成绩大于C02课程中任意一个成绩（即大于最小值） SELECT Sno, Grade FROM SC WHERE Cno = 'C01' AND Grade > ANY ( SELECT Grade FROM SC WHERE Cno = 'C02' ); -- ALL：C01成绩大于C02课程中所有成绩（即大于最大值） SELECT Sno, Grade FROM SC WHERE Cno = 'C01' AND Grade > ALL ( SELECT Grade FROM SC WHERE Cno = 'C02' );
💡 ANY：与子查询结果中任意一个满足条件即可（等价于 > MIN(子查询)）
💡 ALL：与子查询结果中所有值都满足条件（等价于 > MAX(子查询)）
写法等价写法含义
> ANY (子查询)> MIN(子查询)大于子查询最小值
> ALL (子查询)> MAX(子查询)大于子查询最大值
= ANY (子查询)IN (子查询)等于子查询任意值
!= ALL (子查询)NOT IN (子查询)不等于子查询全部值

7️⃣ 相关子查询（EXISTS）

-- 查询选了"C01"课程的学生姓名（EXISTS写法） SELECT Sname FROM Student WHERE EXISTS ( SELECT * FROM SC WHERE SC.Sno = Student.Sno -- 引用外层 Student AND Cno = 'C01' ); -- NOT EXISTS：没有选"C01"的学生 SELECT Sname FROM Student WHERE NOT EXISTS ( SELECT * FROM SC WHERE SC.Sno = Student.Sno AND Cno = 'C01' );
💡 相关子查询：子查询引用了外层查询的表（Student.Sno），随外层逐行执行
💡 EXISTS 只关心子查询是否返回行（有/无），不关心具体值
💡 执行顺序：外层取一行 → 代入内层执行 → 根据EXISTS结果决定保留

✏️ 增 / 修 / 删（DML）及子查询应用

8️⃣ 插入数据 — INSERT

-- 基本插入（指定列） INSERT INTO Student (Sno, Sname, Age, Sex) VALUES ('S06', '吴九', 20, '女'); -- 省略列名（VALUES顺序须与表结构一致） INSERT INTO Student VALUES ('S07', '郑十', 21, '男'); -- 子查询批量插入（SELECT结果插入另一表） -- 注意：SELECT列数、类型须与INSERT目标列匹配 INSERT INTO GoodStudent (Sno, Sname) SELECT Student.Sno, Sname FROM Student, SC WHERE Student.Sno = SC.Sno AND Grade >= 90;
💡 INSERT INTO 表名 (列名...) VALUES (值...) — 指定列插入
💡 省略列名时，VALUES 的顺序和数量必须与建表时的列顺序完全一致
💡 子查询插入：SELECT 的列数和类型必须与目标列一一匹配

9️⃣ 修改数据 — UPDATE

-- 基本修改（指定WHERE条件） UPDATE Student SET Age = 22 WHERE Sno = 'S01'; -- 省略WHERE（修改所有行，慎用！） UPDATE SC SET Grade = Grade * 1.1; -- 利用子查询修改：将选了"C01"的学生年龄+1 UPDATE Student SET Age = Age + 1 WHERE Sno IN ( SELECT Sno FROM SC WHERE Cno = 'C01' );
💡 WHERE 可省略，省略则修改该表所有行（危险操作，需谨慎）
💡 SET 可同时修改多列：SET col1=v1, col2=v2
💡 子查询可用在 WHERE 中，批量更新满足子查询条件的行

🔟 删除数据 — DELETE

-- 基本删除（指定条件） DELETE FROM Student WHERE Sno = 'S05'; -- 省略WHERE（清空表，但保留表结构） DELETE FROM SC; -- 利用子查询删除：删除男生中Grade < 80的选课记录 DELETE FROM SC WHERE Grade < 80 AND Sno IN ( SELECT Sno FROM Student WHERE Sex = '男' );
💡 WHERE 可省略，省略则删除表中所有行（表结构保留，数据清空）
💡 与 DROP TABLE 的区别：DELETE 只删数据，表还在；DROP 连表结构一起删
💡 子查询 DELETE：先在子查询中确定要删的行，再整体执行
📌 DELETE vs TRUNCATE vs DROP：
  DELETE：逐行删，可加 WHERE，可回滚
  TRUNCATE：快速清空整表，不可回滚
  DROP TABLE：连表结构一起删除

1️⃣1️⃣ 连接方式对比

连接类型关键词特点结果行数
等值连接（内连接）JOIN ON / WHERE =只保留两表都有的行≤ 各表行数
自身连接同表两别名表自己和自己连接视条件而定
左外连接LEFT JOIN左表全保留，右表无匹配填NULL≥ 左表行数
笛卡尔积无连接条件所有行的全组合（一般要避免）m × n 行
