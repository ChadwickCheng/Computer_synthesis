🗄️ SQL基础语句详解与示例

📐 DDL — 数据定义语言（CREATE / ALTER / DROP）

1️⃣ 创建表 — CREATE TABLE

CREATE TABLE Student ( Sno VARCHAR(20) NOT NULL, Sname VARCHAR(50), Age INT, Sex VARCHAR(10), PRIMARY KEY (Sno) );
💡 常用数据类型：VARCHAR(n) 变长字符串 / INT 整数 / FLOAT 浮点数
💡 NOT NULL 非空约束 / PRIMARY KEY 主键（唯一 + 非空）
💡 VARCHAR 建议取 10 / 20 / 50 等常见长度

2️⃣ 修改表结构 — ALTER TABLE

-- 改表名 ALTER TABLE Student RENAME TO StudentInfo; -- 增加列 ALTER TABLE Student ADD COLUMN Email VARCHAR(100); -- 删除列 ALTER TABLE Student DROP COLUMN Age; -- 修改列名及类型 ALTER TABLE Student CHANGE COLUMN Sname StudentName VARCHAR(100);
💡 ALTER 只改结构，不影响数据（除 DROP COLUMN 会丢失该列数据）
💡 RENAME / ADD / DROP / CHANGE 四种子命令对应改名、加列、删列、改列

3️⃣ 删除表 — DROP TABLE

DROP TABLE StudentInfo;
⚠️ DROP TABLE 会删除整个表（结构 + 数据），不可恢复！
💡 对比：DELETE FROM Student 只删数据，保留表结构
📌 三者区别：
• DROP TABLE T → 删表（结构+数据全没）
• DELETE FROM T → 删数据（结构保留）
• TRUNCATE TABLE T → 清空数据（结构保留，更快）
🔍 DML — 数据操纵语言（SELECT / INSERT / UPDATE / DELETE）

⚙️ SELECT 执行顺序

① FROM → ② WHERE → ③ GROUP BY → ④ HAVING → ⑤ SELECT → ⑥ ORDER BY → ⑦ LIMIT
💡 写的顺序：SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
💡 执行顺序与书写顺序不同！WHERE 先于 SELECT，所以 WHERE 中不能用 SELECT 定义的列别名
💡 HAVING 在 GROUP 之后执行，可以使用聚集函数；而 WHERE 不行

4️⃣ 基本查询 — SELECT *

-- 查询所有列 SELECT * FROM Student; -- 查询指定列 SELECT Sno, Sname FROM Student;
💡 * 代表所有列；指定列名可只查部分列

5️⃣ 表达式 & 列别名 — AS

-- 表达式计算出生年份，AS 起别名 SELECT Sname, 2026 - Age AS BirthYear FROM Student; -- 表起别名后，SELECT 可用别名引用列 SELECT S.Sname, S.Age FROM Student AS S;
💡 AS 关键字可省略，直接写 列名 别名
⚠️ WHERE 子句中不能使用 SELECT 中定义的列别名（WHERE 先于 SELECT 执行）

6️⃣ 去重 — SELECT DISTINCT

-- 查询出现过的所有院系（去重） SELECT DISTINCT Sex FROM Student; -- 多列去重（两列组合相同才算重复） SELECT DISTINCT Sex, Age FROM Student;
💡 DISTINCT 对结果所有列组合去重，放在 SELECT 后、第一个列名前
🔎 WHERE 条件详解

7️⃣ 比较运算符

SELECT * FROM Student WHERE Age >= 20; -- 支持：= > < >= <= != <> !> !< -- 以及 NOT 取反：NOT Age >= 20
💡 != 和 <> 都表示不等于；NOT 条件 可对任意条件取反

8️⃣ 范围查询 — BETWEEN AND

-- 年龄在 19 到 21 之间（含边界） SELECT * FROM Student WHERE Age BETWEEN 19 AND 21; -- 取反：NOT BETWEEN SELECT * FROM Student WHERE Age NOT BETWEEN 19 AND 21;
💡 等价于 Age >= 19 AND Age <= 21，含两端边界

9️⃣ 集合查询 — IN

-- 查询 S01 或 S03 的学生 SELECT * FROM Student WHERE Sno IN ('S01', 'S03'); -- 取反：NOT IN SELECT * FROM Student WHERE Sno NOT IN ('S01', 'S03');
💡 IN (v1, v2, ...) 等价于 = v1 OR = v2 OR ...，更简洁

🔟 模糊查询 — LIKE

-- % 匹配任意长度字符 SELECT * FROM Student WHERE Sname LIKE '张%'; -- _ 匹配单个字符（如：两字名） SELECT * FROM Student WHERE Sname LIKE '__'; -- 转义：查找名字中含 % 的（用 \ 转义） SELECT * FROM Course WHERE Cname LIKE '数%';
💡 %：任意0+个字符 | _：恰好1个字符
💡 \% 和 \_ 可匹配字面量的 % 和 _

1️⃣1️⃣ 空值查询 — IS NULL

-- 查询未登记成绩的选课记录 SELECT * FROM SC WHERE Grade IS NULL; -- 取反：IS NOT NULL SELECT * FROM SC WHERE Grade IS NOT NULL;
⚠️ NULL 不能用 = NULL 判断，必须用 IS NULL！
💡 NULL 表示"未知/缺失"，任何与 NULL 的比较结果都是 NULL（非真非假）

1️⃣2️⃣ 逻辑运算 — AND / OR / NOT

-- AND：同时满足两个条件 SELECT * FROM Student WHERE Age >= 20 AND Sex = '女'; -- OR：满足任一条件 SELECT * FROM Student WHERE Age = 19 OR Sex = '女'; -- 优先级 AND > OR，括号可改变优先级
💡 优先级：NOT > AND > OR
💡 用括号明确优先级：(A OR B) AND C
📊 聚集函数 & 分组（GROUP BY / HAVING）

1️⃣3️⃣ 聚集函数 — COUNT / SUM / AVG / MAX / MIN

SELECT COUNT(*), -- 统计总行数 COUNT(Grade), -- 统计非NULL行数 SUM(Grade), -- 求和 AVG(Grade), -- 平均值（忽略NULL） MAX(Grade), -- 最大值 MIN(Grade) -- 最小值 FROM SC;
⚠️ 聚集函数只能用在 SELECT 和 HAVING 中，不能用于 WHERE！
💡 COUNT(*) 含NULL行，COUNT(列名) 跳过NULL行
💡 AVG/SUM/MAX/MIN 计算时自动忽略 NULL

1️⃣4️⃣ 分组查询 — GROUP BY

-- 统计每门课程的选课人数和平均分 SELECT Cno, COUNT(Sno) AS StudentCount, AVG(Grade) AS AvgGrade FROM SC GROUP BY Cno;
💡 GROUP BY 把相同值的行归为一组，聚集函数对每组分别计算
💡 SELECT 中未用聚集函数的列，必须出现在 GROUP BY 中

1️⃣5️⃣ 分组筛选 — HAVING

-- 查询选课人数 > 1 的课程 SELECT Cno, COUNT(Sno) AS StudentCount FROM SC GROUP BY Cno HAVING COUNT(Sno) > 1;
💡 HAVING 对分组后的结果筛选，可用聚集函数
💡 WHERE 对原始行筛选（在分组前），不能用聚集函数
📌 口诀：WHERE 筛行，HAVING 筛组
📋 排序与分页（ORDER BY / LIMIT）

1️⃣6️⃣ 排序 — ORDER BY

-- 按成绩降序排列（DESC） SELECT Sno, Cno, Grade FROM SC ORDER BY Grade DESC; -- 多列排序：先按成绩降序，再按学号升序 SELECT Sno, Cno, Grade FROM SC ORDER BY Grade DESC, Sno ASC;
💡 默认升序 (ASC)，降序用 DESC
💡 多列排序中，前面列优先级更高（前列相同时才看后列）
💡 NULL 值在排序中通常排在最前（ASC时）或最后（DESC时），各数据库有差异

1️⃣7️⃣ 分页 — LIMIT

-- 返回前 3 条记录 SELECT * FROM SC ORDER BY Grade DESC LIMIT 3; -- 分页：跳过前 2 条，取 3 条（第2页） SELECT * FROM SC ORDER BY Grade DESC LIMIT 3 OFFSET 2;
💡 LIMIT n：返回前 n 条
💡 LIMIT n OFFSET m：跳过 m 条后取 n 条，用于分页
💡 也可写成 LIMIT m, n（部分数据库支持）
🔐 DCL — 数据控制语言（GRANT / REVOKE）

1️⃣8️⃣ 权限管理 — GRANT / REVOKE

-- 授予用户 alice 对 Student 表的查询权限 GRANT SELECT ON Student TO alice; -- 授予多种权限，并允许再转授 GRANT SELECT, INSERT, UPDATE ON SC TO bob WITH GRANT OPTION; -- 撤销权限 REVOKE SELECT ON Student FROM alice;
💡 GRANT 权限 ON 表 TO 用户：授权
💡 WITH GRANT OPTION：允许被授权用户将权限再转授他人
💡 REVOKE 权限 ON 表 FROM 用户：撤权
📌 常见权限：SELECT / INSERT / UPDATE / DELETE / ALL PRIVILEGES
📋 权限对比：
权限说明
SELECT查询数据
INSERT插入数据
UPDATE修改数据
DELETE删除数据
ALL PRIVILEGES所有权限

1️⃣9️⃣ 增删改 — INSERT / UPDATE / DELETE

-- 插入一行 INSERT INTO Student (Sno, Sname, Age, Sex) VALUES ('S06', '周八', 23, '男'); -- 修改数据 UPDATE Student SET Age = 21 WHERE Sno = 'S01'; -- 删除数据 DELETE FROM Student WHERE Sno = 'S06';
💡 INSERT：可省略列名列表（此时须按表定义顺序给值）
⚠️ UPDATE/DELETE 不加 WHERE 会修改/删除所有行！
💡 DELETE 只删数据，表结构保留；DROP TABLE 连结构一起删
